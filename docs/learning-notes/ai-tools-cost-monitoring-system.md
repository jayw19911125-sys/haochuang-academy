## 🔄 月度更新流程

### 第 1 步：數據收集（每月 1 日）

**收集方式**：
```
1. 訪問每個工具的官網
2. 記錄當前價格
3. 檢查是否有新功能或變化
4. 記錄免費版功能變化
5. 更新 Google Sheets
```

**收集清單**：
```
☐ ShortGenius - 官網 pricing 頁面
☐ CapCut - 官網 pricing 頁面
☐ OpusClip - 官網 pricing 頁面
☐ Klap - 官網 pricing 頁面
☐ Quso - 官網 pricing 頁面
☐ 2Short AI - 官網 pricing 頁面
☐ Synthesia - 官網 pricing 頁面
☐ Runway - 官網 pricing 頁面
☐ Descript - 官網 pricing 頁面
☐ HeyGen - 官網 pricing 頁面
☐ Pictory - 官網 pricing 頁面
☐ Loom - 官網 pricing 頁面
```

### 第 2 步：數據分析（每月 1-2 日）

**分析內容**：
```
1. 計算平均月費
2. 計算成本/支
3. 識別價格變化 > 5% 的工具
4. 識別新功能或功能下降
5. 更新 ROI 計算
6. 生成變化報告
```

**分析公式**：
```
價格變化率 = (新價格 - 舊價格) / 舊價格 × 100%

成本/支 = 月費 / 月產量

ROI = (月產量 × 客戶價值) / 月費

推薦指數 = (功能評分 + 性價比評分 + 穩定性評分) / 3
```

### 第 3 步：決策與通知（每月 2-3 日）

**決策標準**：
```
如果價格變化率 < 5%：
  ✅ 無需調整，保持推薦

如果價格變化率 5-10%：
  ⚠️ 監控，可能需要調整
  - 如果成本/支 > 3：考慮降級推薦
  - 如果成本/支 ≤ 3：保持推薦

如果價格變化率 > 10%：
  🔴 需要立即調整
  - 如果漲價：考慮替代工具
  - 如果降價：提升推薦優先級
```

**通知方式**：
```
1. Slack 通知（子權 + 小鑫）
2. Google Sheets 更新
3. GitHub 版本更新
4. 月度報告
```

**通知模板**：
```
🔔 AI 工具成本更新 - 2026 年 7 月

變化摘要：
- ShortGenius 基礎版：$99 → $99（無變化）
- OpusClip 基礎版：$49 → $49（無變化）
- Klap 基礎版：$29 → $35（↑ 20.7%）⚠️

需要調整的推薦：
- Klap 成本/支從 $0.6 → $0.7（仍在可接受範圍）
- 保持推薦，但監控

新增功能：
- ShortGenius 添加了 AI 字幕生成
- Runway 添加了 4K 導出

已更新：
- Google Sheets
- GitHub docs/learning-notes/ai-tools-cost-comparison.md
```

## 🚨 應急預案

### 應急情況 1：某工具突然漲價 > 20%

**觸發條件**：
- 價格變化率 > 20%
- 成本/支 > 3

**應急響應（24 小時內）**：

**第 1 步：確認信息（1 小時內）**
```
1. 訪問官網確認價格
2. 檢查是否有新功能
3. 聯繫工具客服確認原因
4. 記錄所有信息
```

**第 2 步：評估影響（2 小時內）**
```
1. 計算新的成本/支
2. 比較替代工具
3. 評估對客戶的影響
4. 評估對內部流程的影響
```

**第 3 步：制定對策（4 小時內）**
```
選項 A：接受漲價
- 條件：新功能足以抵消成本增加
- 行動：更新推薦，添加新功能說明

選項 B：降級推薦
- 條件：替代工具成本更低
- 行動：調整推薦優先級，通知客戶

選項 C：停止推薦
- 條件：無可接受的替代方案
- 行動：移除推薦，通知客戶

選項 D：協商
- 條件：我們是高價值客戶
- 行動：聯繫銷售，爭取優惠
```

**第 4 步：執行對策（8 小時內）**
```
1. 更新成本表
2. 更新推薦清單
3. 通知客戶
4. 更新文檔
5. 記錄決策
```

**第 5 步：監控效果（持續）**
```
1. 監控客戶反應
2. 監控替代工具使用情況
3. 定期評估決策效果
4. 必要時調整
```

### 應急情況 2：推薦的工具停止運營

**觸發條件**：
- 工具官網無法訪問
- 工具宣布停止運營
- 工具被收購且功能大幅下降

**應急響應（24 小時內）**：

**第 1 步：確認信息（1 小時內）**
```
1. 多次嘗試訪問官網
2. 檢查官方社交媒體
3. 檢查新聞和公告
4. 聯繫工具客服
```

**第 2 步：評估影響（2 小時內）**
```
1. 有多少客戶在使用？
2. 客戶的工作是否受到影響？
3. 是否有緊急替代方案？
4. 需要多久才能遷移？
```

**第 3 步：制定遷移計劃（4 小時內）**
```
1. 識別最佳替代工具
2. 評估遷移成本
3. 制定遷移時間表
4. 準備遷移指南
```

**第 4 步：通知客戶（8 小時內）**
```
1. 發送緊急通知
2. 解釋情況
3. 提供替代方案
4. 提供遷移支持
```

**第 5 步：執行遷移（持續）**
```
1. 幫助客戶遷移
2. 提供技術支持
3. 監控遷移進度
4. 確保業務連續性
```

### 應急情況 3：某工具功能大幅下降

**觸發條件**：
- 工具移除了關鍵功能
- 工具質量明顯下降
- 工具穩定性問題

**應急響應（24-48 小時內）**：

**第 1 步：確認問題（2 小時內）**
```
1. 自己測試工具
2. 收集客戶反饋
3. 檢查官方公告
4. 聯繫工具客服
```

**第 2 步：評估影響（4 小時內）**
```
1. 問題是否影響核心功能？
2. 是否有臨時解決方案？
3. 工具是否會恢復？
4. 是否需要立即替換？
```

**第 3 步：制定應對方案（8 小時內）**
```
選項 A：等待修復
- 條件：工具承諾快速修復
- 行動：通知客戶，提供臨時方案

選項 B：降級推薦
- 條件：問題可能長期存在
- 行動：調整推薦，提供替代工具

選項 C：停止推薦
- 條件：問題嚴重且無法修復
- 行動：移除推薦，通知客戶
```

**第 4 步：執行對策（24 小時內）**
```
1. 通知客戶
2. 提供替代方案
3. 更新文檔
4. 記錄決策
```

### 儀表板自動化

**使用 Google Sheets 公式自動計算**：
```
=IF(B2>B3, "↑", IF(B2<B3, "↓", "→"))  // 價格變化趨勢

=(B2-B3)/B3*100%  // 價格變化率

=IF(F2>0, G2/F2, "N/A")  // 成本/支

=IF(H2>3, "🔴", IF(H2>2, "⚠️", "✅"))  // 成本警告
```
