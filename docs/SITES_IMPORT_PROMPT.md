# 好創學院 2.1｜ChatGPT Sites 匯入與建站總指令

@Sites

請從 GitHub Repository `jayw19911125-sys/haochuang-academy` 的分支 `feat/academy-v2-gpt-sites-ready` 建立一個名為「好創學院」的私有 ChatGPT Site。

## 執行方式

1. 先讀取並檢查整個分支，不要只看 README。
2. 以 `/v2` 的 `AcademyV2Corrected` 為新版產品基礎；舊 `/` 只能作視覺與功能參考，不得把舊 14 章、30 天里程碑、XP、排行榜與 deviceId 身份制度帶入新站。
3. 檢查現有 Vite／React 專案是否能產生 Sites 相容部署產物；若不相容，請保留內容、資訊架構與視覺語言，改造成 Sites 推薦的相容架構。
4. 先建立「已儲存版本／私有預覽」，不要直接公開部署。完成後回報預覽狀態、相容性修改、資料庫 Migration 與尚未處理風險。

## Site 定位

好創學院不是一般 LMS，也不是長篇新人教材。它是好創內部的：

- 職務學習站
- 工作操作站
- 工作證據提交站
- 主管評核站
- 1／3／6 個月能力驗證站

首頁主張維持：

> 不是看完教材，是要完整跑通案件。

## 存取與身份

- 站點預設只允許 Owner 與指定好創成員存取，不得公開。
- 使用 workspace-authenticated user identity。
- 伺服器端以 `oai-authenticated-user-email` 作為唯一人員識別，`oai-authenticated-user-full-name` 僅作顯示名稱。
- 所有授權判斷必須在伺服器端完成。
- 移除 deviceId、分享識別碼與可由前端偽造的身份。
- 建立管理員白名單或角色表；Dennis／指定管理者才能查看全員資料與做主管評核。

## D1 資料庫

使用 D1 綁定名稱 `DB`。採用分支內 `sites/migrations/0001_academy.sql` 作為起始 Schema，必要時可以調整，但必須保留下列資料能力：

- 使用者與職務
- 能力模組進度
- 工作證據連結
- 第 1、3、6 個月考核狀態
- 主管回饋與核准紀錄
- 內容／模組版本
- 稽核時間與最後更新人

不要把以下資料複製到 D1：

- 客戶帳密、API Key、Token
- 正式合約、發票、銀行與財務明細
- 原始影片素材與大型成品
- Monday 任務的完整複本
- 未脫敏客戶機密

## 必須完成的前台區域

### 1. 我的工作學習台

顯示：使用者姓名與職務、所在階段、尚未完成的能力模組、下一個建議行動、尚缺工作證據、最近主管回饋、最新更新規則／教材。不得以學習時間、XP 或排行榜當能力。

### 2. 職務與權責

呈現 CEO、COO、商業內容企劃／社群營運、拍攝剪輯／內容交付四個核心職務的職務使命、最終負責結果、可自行決定、必須往上回報與工作完成定義。

### 3. 企劃養成路徑

強制持續顯示自主學習能力與文書行政能力。第 1 個月必須完整呈現並跑通分支資料定義的 13 階段企劃流程：

1. 接案資訊與服務範圍確認
2. 市場調查與產業理解
3. 客戶商業診斷
4. 受眾診斷與購買旅程
5. 競品與標竿分析
6. 全案策略與單線策略判斷
7. 平台角色與內容分工
8. 六平台一個月內容矩陣與排程
9. 圖文、輪播、文案、Threads、LINE OA、短影音製作
10. 內部審核、客戶核准與版本控制
11. 發布、互動與社群營運
12. 數據回顧與下一輪優化
13. 知識沉澱、AI 專案與 SKILL 資產化

每一階段要顯示目的、工具、產出、Definition of Done。

### 4. 1／3／6 個月能力驗證

員工可見正式名稱只能是：

- 第 1 個月｜全流程獨立執行
- 第 3 個月｜策略整合與專業進階
- 第 6 個月｜多案主責與穩定交付

不得出現「比其他公司強」、「高於一般公司」等員工比較型名稱。考核採證據制；每一項支援未開始、進行中、待審核、通過、需改善，需改善必須有原因、改善要求與期限。

### 5. 工作證據提交

每個能力模組允許提交證據標題、正本連結（Monday／Notion／Drive／NAS 等）、證據類型、說明、所屬月份與能力模組。Site 只保存連結與評核資料，不複製正式檔案。

### 6. 主管檢視

管理者可以查看成員階段、未完成能力與證據、審核證據、填寫回饋、設定改善期限、判定 1／3／6 個月結果、查看歷史紀錄。不得只按學習完成數判定能力。

### 7. 工作武器庫

維持正本分工：Monday 管 CRM／案件／任務；Notion 管 SOP／知識；Drive／NAS 管正式文件與素材；Slack 管討論與卡點；LINE OA 是客戶窗口；GitHub 管程式、AI 規則、Schema 與版本；好創學院只管入口、互動、學習、證據與評核。

## UI／UX 要求

- 沿用好創橘色品牌主色與深色工作站視覺，但更乾淨、可掃讀。
- 桌機優先並完成手機 RWD。
- 左側導覽必須真的能切換區域，不得使用失效 anchor。
- 不要做一頁超長 14 章教材。
- 不要放排行榜、XP、紙屑動畫或與工作能力無關的遊戲化。
- 流程區用時間軸／工作流，考核區用證據清單與狀態表。
- 所有互動具備 loading、empty、error、success 狀態。
- 支援鍵盤、可見 focus、足夠對比與 reduced motion。

## 資料與安全要求

- 所有 D1 寫入由伺服器端根據登入身份決定使用者，不接受前端任意 email。
- 管理員 API 必須在伺服器端驗證角色。
- 不將 Secret 寫進程式碼、Prompt、附件或 `.openai/hosting.json`。
- 不收集身分證、銀行、薪資、健康資訊或付款卡資料。
- Site 保持私有，直到 Dennis 驗收後才調整分享範圍。

## 完成條件

不要公開部署；先保存版本並讓我審查。
